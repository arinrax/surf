# 🍳 Mock-бэкенд «Кулинарная студия»

**Локальный mock-бэкенд на Python + SQLite**, эмулирующий работу реального сервера для клиентского мобильного приложения сети кулинарных студий.  
Проект реализует MVP-функционал без внешних зависимостей — только стандартная библиотека Python.

> **Дата актуализации:** 6 июля 2026 г.  
> **Скоуп:** MVP-сущности (`Clients`, `Slots`, `Bookings`, `Equipment`, `Chefs`).  
> **Вне скоупа:** оплата, отзывы, админка, интерфейсы шефа/администратора.

## 🔌 API-контракт

### `GET /api/v1/slots`
Возвращает список активных слотов на указанный диапазон дат (по умолчанию — 7 дней).

**Query-параметры:**
- `start_date` (optional, ISO 8601, default: today)
- `end_date` (optional, ISO 8601, default: today + 6 days)

**Response (200 OK):**
```json
{
  "slots": [
    {
      "id": 1,
      "start_time": "2026-07-07T18:00:00",
      "menu_name": "Итальянская классика",
      "chef": { "id": 1, "name": "Марко Росси", "rating": 4.9 },
      "capacity": 12,
      "free_seats_count": 10,
      "is_rent_available": true,
      "status": "active"
    }
  ]
}
```

### `POST /api/v1/bookings`
Создаёт бронь с атомарной проверкой мест и проката.

**Request Body:**
```json
{
  "client_id": 101,
  "slot_id": 1,
  "equipment_type": "rental",
  "allergies": "Глютен, орехи"
}
```

**Response (201 Created):**
```json
{ "id": 501, "status": "confirmed", "message": "Бронь успешно создана" }
```

**Ошибки (400 Bad Request):**
| Код | Условие | Сообщение |
|---|---|---|
| `SLOT_FULL` | Мест нет | «На этот класс только что записались. Выберите другое время» |
| `RENT_EXHAUSTED` | Прокат закончился | «Все прокатные наборы разобраны. Приходите со своей экипировкой» |

### `DELETE /api/v1/bookings/{id}`
Отменяет бронь клиентом.

**Response (200 OK):**
```json
{ "status": "cancelled_by_client", "message": "Бронь успешно отменена" }
```

**Ошибки (400 Bad Request):**
| Код | Условие | Сообщение |
|---|---|---|
| `CANCELLATION_TOO_LATE` | До начала ≤ 10 мин | «Продукты уже закуплены. Свяжитесь с нами по телефону» |
| `BOOKING_NOT_FOUND` | Бронь не найдена | «Бронь не найдена» |
| `ALREADY_CANCELLED` | Повторная отмена | «Бронь уже отменена» |

## 🧠 Ключевые бизнес-правила

1. **Атомарность бронирования** — транзакция SQLite защищает от овербукинга и race conditions.
2. **Правило 10 минут** — отмена невозможна менее чем за 10 минут до начала слота (продукты закуплены).
3. **Empty State** — слоты со статусом `cancelled_by_studio` не возвращаются в общем списке.
4. **Source of Truth** — серверное время используется для всех временных проверок (защита от обхода через перевод часов на устройстве).
5. **Лимит мест** — строго 8 или 12 мест на слот (CHECK-ограничение в схеме).
6. **Фонд проката** — отдельный для каждого слота, учитывается атомарно с бронью.
7. **Мягкое удаление** — брони не удаляются физически, статус меняется на `cancelled_by_client` (сохранение истории).
8. **Гарантия от двойных броней** — клиент не проверяет места локально, всё решает сервер.
