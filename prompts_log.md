## Методы промптинга, которые я использовала

1. **Grounding (Заземление)** — в промптах 1.1 и 2.1 явно указывала ИИ «не выдумывать того, чего нет в домене» и «опираться строго на текст».
2. **Ролевое моделирование** — задавала ИИ конкретную роль: «системный аналитик», «бэкенд-архитектор», «Python-разработчик», «Senior QA-инженер».
3. **Итеративность** — не пыталась получить всё ТЗ одним промптом. Шла последовательно: бриф → домен → User Stories → экраны → БД → код.
4. **Мультиагентный подход** — для ревью ТЗ открывала новый чат с «чистым контекстом», чтобы ИИ посмотрел свежим взглядом.
5. **Формализованные баг-репорты** — для исправления багов использовала формат писем QA-инженера разработчику (как в реальной команде), что улучшило качество фикса.
6. **Сначала ширина, потом глубина** — при генерации тест-кейсов сначала попросила план покрытия (таблица), а только потом — детальные JSON-кейсы.
